# Make Product Releases Announced at Waves '25: AI Agents and Maia

## Strategic Framework: Build, Accelerate, Scale

Make introduced a new product philosophy at its Waves '25 conference centered on three pillars: **Build**, **Accelerate**, and **Scale**. VP of Product Anton Danilov emphasized that "visual-first approach gives you ultimate clarity and control over your automation and AI."

Make positions itself as "the only automation and AI platform on the market" offering complete visibility into real-time business operations — combining deterministic automation with adaptive AI agents simultaneously.

---

## Build Pillar Features

### Maia by Make (Waitlist Available)
The standout announcement is a conversational AI builder integrated directly into the platform. Key characteristics:
- "Maia is your personal AI and automation builder"
- Enables users to "generate scenarios with AI end-to-end" through natural language prompts
- Promises speed with "full visual transparency" — users retain the ability to clarify decisions during the generation process
- Designed to lower the barrier to creating complex multi-step scenarios without deep technical knowledge

### Workflow Infrastructure Improvements
- **If–Else and Merge modules** (early 2026): Enable conditional logic branching and path consolidation within scenarios — a long-requested feature for cleaner flow control
- **Scenario Run Replay** (available now): Allows re-running scenarios with historical data, significantly improving debugging workflows
- **Scenario Sharing** (coming soon): One-click deployment with secure sharing links for collaboration
- **Subscenarios** (available now): Transform individual scenarios into reusable functions or callable APIs — enabling modular automation architecture

---

## Accelerate Pillar Features

### Next-Generation AI Agents (Waitlist Available)
AI Agents are built directly into the scenario canvas with real-time visualization of agent reasoning steps. Features include:
- File support within agent interactions
- Conversation history for multi-turn agent workflows
- Prompt improvement tools for iterative agent refinement
- Visual display of reasoning steps as agents execute — providing transparency into AI decision-making

### Agent Enhancements
- **Library of expert-built agents** (coming soon): Pre-built agents for common business workflows, deployable without configuration
- **Agent sharing capability** (coming soon): Share agents across teams or externally
- **Module Tools**: Convert any existing Make module into a callable tool for AI agents — enables agents to trigger any of Make's 3,000+ integrations
- **Make AI Provider**: Native LLM access without requiring external API keys or separate AI service subscriptions
- **Make Code**: Execute JavaScript or Python within scenarios; Enterprise tier supports third-party library imports

---

## Scale Pillar Features

### Make Grid (Now Available to All Paid Plan Users)
A comprehensive visualization tool providing a map of an organization's entire automation landscape:
- Supports 40+ app connectors with real-time status
- Real-time credit and data consumption monitoring across all scenarios
- Automated issue highlighting to surface failures and bottlenecks proactively
- Snapshot sharing for cross-functional collaboration and stakeholder reporting

### Integration and Security
- **MCP Server/Client**: Connect Make scenarios as callable tools for AI agents and external services via the Model Context Protocol — enabling Make to serve as an action layer for external AI systems
- **Scenario Rate Limits**: Prevent disruptive data spikes from overwhelming connected services
- **Authenticated Webhooks**: Systemic webhook authentication for improved security posture
- **Credits per Teams** (Enterprise): Granular resource allocation control at the team level

---

## Strategic Significance

The Waves '25 announcements reflect Make's pivot toward positioning the platform as a complete AI operations layer, not just a workflow connector:

1. **Maia** moves scenario creation from technical configuration to natural language conversation
2. **AI Agents with visual reasoning** address the enterprise concern about AI opacity — users can see what the agent is doing in real time
3. **Make Grid** provides organizational-level visibility that enterprise buyers require before deploying automation at scale
4. **MCP integration** signals Make's intent to be the execution layer for external AI systems (LLMs, custom agents) — not just a standalone automation platform

The combination of deterministic scenario automation and adaptive agentic AI within a single visual interface is Make's core differentiator heading into 2026.
