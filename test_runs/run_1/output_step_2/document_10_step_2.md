# Make Product Releases at Waves '25 — AI Agents and Maia

Make's Waves '25 conference introduced a **Build / Accelerate / Scale** product framework, positioning the platform as a complete AI operations layer combining deterministic automation with adaptive agentic AI.

**Build pillar:**
- *Maia by Make* (waitlist): Conversational AI builder; generates scenarios end-to-end from natural language prompts with full visual transparency and user clarification during generation.
- *If–Else and Merge modules* (early 2026): Conditional branching and path consolidation — long-requested for cleaner flow control.
- *Scenario Run Replay* (available): Re-run scenarios with historical data for debugging.
- *Subscenarios* (available): Transform scenarios into reusable functions or callable APIs — enables modular automation architecture.
- *Scenario Sharing* (coming soon): One-click deployment via secure sharing links.

**Accelerate pillar:**
- *Next-Generation AI Agents* (waitlist): Built into the scenario canvas with real-time reasoning step visualization, file support, conversation history, and prompt improvement tools.
- *Library of expert-built agents* (coming soon): Pre-built agents for common workflows, deployable without configuration.
- *Module Tools*: Convert any Make module into a callable tool for AI agents — unlocks all 3,000+ integrations as agent actions.
- *Make AI Provider*: Native LLM access without external API keys.
- *Make Code*: JavaScript or Python execution; Enterprise supports third-party library imports.

**Scale pillar:**
- *Make Grid* (available to all paid plans): Organization-wide automation map — 40+ connectors, real-time credit/data monitoring, automated issue highlighting, snapshot sharing.
- *MCP Server/Client*: Make scenarios callable as tools by external AI systems; enables Make as an action layer for external LLMs and agents.
- *Scenario Rate Limits*: Prevents data spikes from overwhelming connected services.
- *Authenticated Webhooks*: Systemic webhook authentication.
- *Credits per Teams* (Enterprise): Team-level resource allocation.
